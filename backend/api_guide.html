1. Introduction

Welcome to the **AI Image Detector Developer Portal**. Our multi-signal forensic analysis API provides enterprise-grade detection capabilities, enabling developers to identify AI-generated and deepfake media at scale. By combining pixel-level forensics with state-of-the-art vision models, it remains robust against common platform modifications, including metadata removal, recompression, resizing, and screenshots (typical of sharing on Telegram, WhatsApp, Instagram, or Facebook).

Dual-Path Classification

Parallel pipelines to detect fully AI-generated images (Midjourney, DALL-E, Flux) and face manipulations (deepfakes).

Compression Robustness

Adaptive weights automatically shift from metadata to semantic features (CLIP/ViT) when social media sharing strips headers.

Granular Breakdown

Returns ELA compression, fast Fourier transforms (FFT), LBP texture anomalies, and camera authenticities.

Asynchronous Processing

Support for background jobs on large or high-resolution images to prevent standard HTTP timeouts.

Authentication

All requests must be authenticated using an API Key. Include your key in the request headers: 

X-API-Key: your_secure_api_key_here

2. Quick Start Guide

Integrate AI detection into your application in three simple steps:

  1. Obtain your API Key from your developer portal dashboard.
  2. Make a POST request to the `/api/v2/analyze` endpoint with the image file in `multipart/form-data`.
  3. Process the JSON verdict to automatically tag, moderate, or alert your backend logic.
cURL Example Request 
curl -X POST "http://localhost:8000/api/v2/analyze" \
  -H "X-API-Key: your_secure_api_key_here" \
  -F "file=@sample_image.jpg"

3. API Endpoints

POST /api/v2/analyze Core Image Forensics Engine

Submit a JPG, PNG, or WEBP image to run a full multi-layer analysis.

Parameter Type Required Description
file Binary Yes Multi-part form field containing raw image binary.
async_mode Query Bool No Instantly return `job_id` and process in background.
GET /api/v2/jobs/{job_id} Poll Async Background Job

Poll the state of a large background file execution. Returns completed standard payload or a processing state.

HeaderRequiredDescription
X-API-KeyRecommendedAPI key for production deployments.

Status Codes

  • 200 — Analysis completed successfully.
  • 202 — Job accepted (when async_mode=true).
  • 400 — Invalid file type or payload.
  • 401 — Missing or invalid API key.
  • 413 — File exceeds 10MB limit.
  • 500 — Internal analysis failure.
POST /api/analyze Legacy v1-compatible response

Same engine as v2; returns simplified is_ai_generated, confidence, and signals for older integrations.

5. SDK Usage & Integration

Installation

Use any HTTP client. Optional Python helper:

pip install requests

Authentication

Set X-API-Key on every request. Local development may run without a key; production should enforce keys via your API gateway.

Uploading Images & Reading Results

  1. POST multipart file to /api/v2/analyze.
  2. Read verdict: AI_GENERATED, DEEPFAKE, or AUTHENTIC.
  3. Use dual_detection.ai_generated.score and dual_detection.deepfake.score for parallel GenAI vs face-manipulation signals.
  4. Check analysis_context.messenger_shared when the image was likely re-compressed by Telegram/WhatsApp.
  5. Use breakdown.compression_robust_score for spectral forensic confidence after social sharing.

Error Handling

Retry 5xx with exponential backoff. Do not retry 4xx except 429 (rate limit). Parse detail from FastAPI error bodies.

Rate Limiting

Default: 60 requests/minute per API key (configurable on enterprise plans). Response headers: X-RateLimit-Limit, X-RateLimit-Remaining, Retry-After on 429.

API Versioning

Current stable: v2 (/api/v2/*). Legacy /api/analyze remains supported and maps to v2 internally. Breaking changes increment the path version only.

Best Practices

  • Send the highest quality copy available; avoid re-saving PNG→JPEG→PNG chains.
  • For Telegram/WhatsApp workflows, analyze the downloaded file as users receive it.
  • Use async_mode=true for images above 4MP or batch moderation pipelines.
  • Combine GenAI + deepfake scores when moderating user avatars vs full-scene synthetic art.

6. Frequently Asked Questions

How does the API handle messaging compressed JPEGs from Telegram/WhatsApp?

When an image is shared via messaging apps, EXIF and C2PA are stripped and JPEG is re-quantized. The API enters messenger context, runs compression-robust spectral forensics plus multi-branch HF inference (including synthetic re-JPEG test views), and fuses scores without over-penalizing when semantic models alone read as "real."

Are my images saved on your servers?

No. Uploaded images are processed strictly in-memory during analysis and are immediately deleted afterwards. No media binaries are stored or used to retrain models without explicit enterprise opt-in.

8. Security & Webhooks

Security Recommendations

  • Rotate API keys quarterly; never embed keys in client-side JavaScript.
  • Terminate TLS at your edge; use HTTPS only in production.
  • Images are processed in-memory and not retained by default.

Webhooks (Enterprise)

Optional callback_url on async jobs delivers a signed POST when analysis completes. Verify X-Signature-Hmac-Sha256 with your webhook secret.

9. Terms of Service

Detection scores are probabilistic forensic estimates, not legal proof. Use human review for high-stakes decisions. By using the API you agree to acceptable-use policies prohibiting harassment, non-consensual imagery, and unlawful surveillance.
